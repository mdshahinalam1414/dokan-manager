# দোকান ম্যানেজার (Dokan Manager) — সম্পূর্ণ ফ্রি সেটআপ গাইড

এই সিস্টেমে **কোনো টাকা খরচ হবে না**। হোস্টিং হবে GitHub Pages এ, আর ডেটাবেস হবে আপনার নিজের একটা Google Sheet।

---

## ধাপ ১: Google Sheet + Apps Script (ব্যাকএন্ড/ডেটাবেস)

1. [sheets.google.com](https://sheets.google.com) এ গিয়ে একটা নতুন **Blank Spreadsheet** খুলুন। নাম দিন যেমন "Dokan Database"।
2. উপরে মেনু থেকে **Extensions > Apps Script** এ ক্লিক করুন।
3. যেই কোড এডিটর খুলবে, ওখানে থাকা সব ডিফল্ট কোড মুছে ফেলুন।
4. এই প্রজেক্টের `apps-script/Code.gs` ফাইলের পুরো কোড কপি করে পেস্ট করুন।
5. উপরে **Save** (💾 আইকন) ক্লিক করুন।
6. ফাংশন ড্রপডাউন (Run বাটনের পাশে) থেকে **`setup`** সিলেক্ট করে **Run** চাপুন।
   - প্রথমবার একটা পারমিশন চাইবে — **Review permissions > আপনার Google একাউন্ট সিলেক্ট > Advanced > Go to project (unsafe) > Allow** করুন। (এটা নিরাপদ, কারণ এটা আপনার নিজেরই স্ক্রিপ্ট।)
   - Run করার পর আপনার Sheet এ অনেকগুলো ট্যাব (Capital, Sales, Inventory ইত্যাদি) তৈরি হয়ে যাবে।
7. **নতুন ভার্সনে পাসওয়ার্ড সিস্টেম পাল্টেছে** — এখন `setPassword()` ফাংশনের বদলে `setup()` চালালেই একটা ডিফল্ট Owner ইউজার তৈরি হয়ে যায়:
   - **ইউজারনেম**: `owner`
   - **পাসওয়ার্ড**: `owner123`

   সাইটে প্রথমবার এই দিয়ে লগইন করে সাথে সাথে **স্টাফ/ইউজার** ট্যাব থেকে নতুন পাসওয়ার্ড সেট করা ইউজার যোগ করুন, বা Google Sheet এর `Users` ট্যাবে গিয়ে সরাসরি `owner` এর পাসওয়ার্ড বদলে দিন।

### Web App হিসেবে ডিপ্লয় করুন

9. উপরে ডানপাশে **Deploy > New deployment** ক্লিক করুন।
10. গিয়ার আইকনে ক্লিক করে টাইপ **Web app** সিলেক্ট করুন।
11. সেটিংস দিন:
    - **Execute as**: Me (আপনার একাউন্ট)
    - **Who has access**: Anyone
12. **Deploy** ক্লিক করুন, আবার পারমিশন চাইলে Allow করুন।
13. যেই **Web app URL** দেখাবে (এমন দেখতে: `https://script.google.com/macros/s/xxxxxxx/exec`), সেটা কপি করে রাখুন — এটাই আপনার API URL।

> ⚠️ ভবিষ্যতে `Code.gs` এ কোনো পরিবর্তন করলে, শুধু Save করলেই হবে না — **Deploy > Manage deployments > ✏️ Edit > New version > Deploy** করতে হবে, নাহলে পুরোনো কোড চলতেই থাকবে।

## ধাপ ২: ফ্রন্টএন্ড সাইট (GitHub Pages)

1. [github.com](https://github.com) এ একটা একাউন্ট খুলুন (না থাকলে)।
2. একটা নতুন **Public repository** তৈরি করুন, নাম দিন যেমন `dokan-manager`।
3. এই প্রজেক্টের ৪টা ফাইল আপলোড করুন (repository তে "Add file > Upload files"):
   - `index.html`
   - `style.css`
   - `app.js`
   - `config.js`
4. `config.js` ফাইলটা GitHub এ ওপেন করে **Edit (✏️)** করুন, এবং
   ```js
   const API_URL = "PASTE_YOUR_APPS_SCRIPT_WEB_APP_URL_HERE";
   ```
   এই লাইনে, `PASTE_YOUR_APPS_SCRIPT_WEB_APP_URL_HERE` এর জায়গায় ধাপ ১.১৩ এ পাওয়া Web App URL টা বসান (কোটেশনের ভিতরে)। **Commit changes** করুন।
5. রিপোজিটরির **Settings > Pages** এ যান।
6. **Branch**: `main` (বা `master`) সিলেক্ট করে **Save** করুন।
7. কিছুক্ষণ পর একটা লিংক পাবেন, যেমন:
   `https://your-username.github.io/dokan-manager/`
   — এটাই আপনার সাইটের ঠিকানা। এই লিংক আপনি নিজে বা যাকে চান শেয়ার করতে পারবেন; পাসওয়ার্ড ছাড়া কেউ ডেটা দেখতে পারবে না।

## ব্যবহার

- সাইটে ঢুকে ইউজারনেম ও পাসওয়ার্ড দিয়ে লগইন করুন (প্রথমবার: `owner` / `owner123`)।
- উপরে ডানে **EN/বাং** বাটনে ভাষা পরিবর্তন করা যাবে।
- **সেটিংস** ট্যাবে গিয়ে দোকানের নাম, ঠিকানা, ফোন নাম্বার দিন — এটা ভাউচারে প্রিন্ট হবে।
- **ইনভেন্টরি** ট্যাবে প্রোডাক্ট যোগ করুন (লো-স্টক থ্রেশহোল্ড সহ), তারপর **স্টক ক্রয়** দিয়ে স্টক ইন করুন। স্টক থ্রেশহোল্ডের নিচে নামলে ড্যাশবোর্ডে লাল এলার্ট দেখাবে।
- **বিক্রয়** ট্যাবে প্রোডাক্ট বেছে বিক্রি করলে ডিসকাউন্ট, ভ্যাট % এবং ক্যাশ+ব্যাংক আলাদা আলাদা (split) পেমেন্ট দেওয়া যাবে। অটোমেটিক ভাউচার তৈরি হয়ে প্রিন্ট প্রিভিউ আসবে, এবং যেকোনো সময় ভাউচার নাম্বার দিয়ে আবার প্রিন্ট করা যাবে।
- বাকিতে বিক্রি করলে বা বাকিতে মাল কিনলে, তা অটোমেটিক **দেনাদার-পাওনাদার** ট্যাবে চলে যাবে, এবং **কাস্টমার/সাপ্লায়ার** ট্যাবে সেই পার্টির সম্পূর্ণ ইতিহাস দেখা যাবে।
- **রিপোর্ট** ট্যাবে যেকোনো তারিখের রেঞ্জের লাভ-লোকসান রিপোর্ট, মাসিক ট্রেন্ড চার্ট এবং প্রিন্ট করার সুবিধা আছে।
- **স্টাফ/ইউজার** ট্যাবে (শুধু Owner দেখবে) নতুন স্টাফ লগইন তৈরি করা যাবে। স্টাফরা লগইন করলে শুধু বিক্রয় ও ইনভেন্টরি (ভিউ) দেখতে পারবে — মূলধন, রিপোর্ট, সেটিংস ইত্যাদি সংবেদনশীল তথ্য দেখতে পারবে না।

## যা মাথায় রাখবেন

- **ডেটা কোথায় থাকে**: আপনার নিজের Google Sheet এ, যা আপনি নিজের একাউন্টে যেকোনো সময় দেখতে/এক্সপোর্ট করতে পারবেন (Google Drive এ)।
- **সিকিউরিটি**: পাসওয়ার্ড ছাড়া কেউ ডেটা এক্সেস করতে পারবে না, কারণ চেকটা সার্ভার সাইডে (Apps Script) হয়। তবু, GitHub রিপোজিটরি চাইলে ভবিষ্যতে Private ও করতে পারবেন (Settings > Danger Zone) — GitHub Pages ফ্রি প্ল্যানে Private repo থেকেও Pages কাজ করে যদি GitHub Pro/Team থাকে; একদম ফ্রি একাউন্টে Public repo রাখতে হবে (কোড পাবলিক থাকবে, কিন্তু ডেটা না)।
- **ব্যাকআপ**: মাঝে মাঝে Google Sheet টা **File > Make a copy** করে রাখলে বাড়তি নিরাপত্তা থাকবে।

কোনো ধাপে আটকে গেলে বা কোনো ফিচার/পরিবর্তন লাগলে জানাবেন।
